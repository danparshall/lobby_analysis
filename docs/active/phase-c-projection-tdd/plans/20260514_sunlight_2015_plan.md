# Sunlight 2015 Projection Implementation Plan

**Goal:** Ship `project_sunlight_2015(state, cells)` — a per-state projection function that reads 13 v2 compendium cells and produces Sunlight 2015's 4 in-scope per-item scores, validated against the published per-state CSV.

**Originating conversation:** [`../convos/20260514_rubric_plans_drafting.md`](../convos/20260514_rubric_plans_drafting.md) (Sub-0 of 5 — playbook gap audit + data-year audit). This plan was drafted in Sub-1 (Stream 1 plans).

**Context:** Rubric #3 in Phase C's locked rubric order (after CPI 2015 C11 + PRI 2010, both shipped). Sunlight is a **shallow** rubric whose value is **cross-rubric redundancy on heavily-shared rows** (11 of 13 rows touched are read by ≥1 other rubric), not novel observables. Sunlight is also Stream 1's lead: its α form-type split introduces 6 bill_id/position cells that Opheim 1991 (rubric #6, second Stream-1 plan) reuses via β AND-projection. **Sunlight must land before Opheim's plan can reference concrete cell IDs.**

**Confidence:** High on per-item mappings (spec doc factual-audit-clean; α/β/γ conventions locked 2026-05-07 and 2026-05-11 in archived `compendium-source-extracts`). Lower on data year (MEDIUM-LOW per `../results/20260514_rubric_data_years.md` — paper methodology re-read is a Phase 0 task).

**Architecture:** Function-per-item dispatcher. All 4 in-scope items have bespoke compound logic (nested-tier tables, OR projections, typed-cell IS-NOT-NULL with sign convention) — declarative table would not compress them cleanly. Mirrors `src/lobby_analysis/projections/cpi_2015_c11.py`'s pattern. **No aggregation function** — Sunlight's published `Total` and `Grade` cannot be reproduced (item 4 excluded per audit decision); the projection produces 4 per-item scores and nothing else.

**Branch:** `phase-c-projection-tdd` (worktree at `.worktrees/phase-c-projection-tdd`).

**Tech Stack:** Python 3.12, Pydantic v2, pytest, `uv` for env management. `load_v2_compendium()` from `lobby_analysis.compendium_loader` for cell access.

## Scope qualifier — item 4 excluded

**Excluded:** Sunlight item 4 (`document_accessibility`, 5-tier `−2 to 2`).

**Reason:** Audit decision 2026-05-07 (see [`docs/historical/compendium-source-extracts/results/projections/20260507_sunlight_atomic_audit.md`](../../../historical/compendium-source-extracts/results/projections/20260507_sunlight_atomic_audit.md) "User decision (2026-05-07 pm)" section). The 5-tier ordinal conflates 3–4 sub-features (digital filing, registration form online, expenditure form online, blank forms online) and the `−1/−2` tier descriptors are a documented near-typo ("either" vs "both" with logically inverted semantics). The cell→tier function is not well-defined.

**In scope:** Sunlight items 1 (Lobbyist Activity), 2 (Expenditure Transparency), 3 (Expenditure Reporting Thresholds), 5 (Lobbyist Compensation).

**Max reproducible:** 4 per-item scores. Sunlight's published `Total` (arithmetic sum across all 5 items) and `Grade` (letter, empirically-derived cutoffs) **cannot be reproduced**. Per-item validation only.

**STOP clause:** If a borderline item surfaces (analogous to FOCAL-1 — user pulled `revolving_door.1` IN as a registration-form disclosure observable on 2026-05-13), STOP and ask the user before pulling item 4 IN. The audit decision is recent and explicit; do not relitigate without a user-surfaced reason.

## Validation regime — Strong (per-state per-item)

**Tier:** Strong — per-state per-atomic-item ground truth published.

**Ground truth:** [`papers/Sunlight_2015__state_lobbying_disclosure_scorecard_data.csv`](../../../../papers/Sunlight_2015__state_lobbying_disclosure_scorecard_data.csv). 50 states × 4 in-scope item columns (`Lobbyist Activity`, `Expenditure Transparency`, `Expenditure Reporting Thresholds`, `Lobbyist Compensation`) = **200 per-cell ground-truth values**.

**Caveat — footnote markers.** Per-state CSV cells carry footnote markers (`*`, `**`, `***`, `^`, `^^`) on individual values; meaning undocumented (flagged in [`docs/historical/compendium-source-extracts/results/items_Sunlight.md`](../../../historical/compendium-source-extracts/results/items_Sunlight.md) §7). Strip markers before integer-coercion comparison; flag marker-carrying cells as caveat'd validation points (write the marker presence into the test failure messages so they're audit-traceable).

**No end-to-end aggregation check.** Item 4 exclusion forces this. The aggregation test slot in the standard test triplet (`test_<rubric>_aggregation.py`) instead verifies the projection module **does not** offer a `project_sunlight_total` or `project_sunlight_grade` function — implementing one would smuggle a wrong-by-design API into the module.

**Validation tolerance:** `==` exact match per cell (signed integer tier values). The 4 in-scope items publish signed integer tiers; no rounding artifacts to admit.

### Item 1 — `sunlight_2015.lobbyist_activity` (4-tier, −1 to 2)

**Helper:** `project_sunlight_item1(cells: dict) -> int` returns `−1 | 0 | 1 | 2`.

**Reads 6 binary cells** (α form-type split: 3 reg-form-side × 3 spending-report-side):

| v2 row id | axis |
|---|---|
| `lobbyist_reg_form_includes_general_subject_matter` | `legal_availability` |
| `lobbyist_reg_form_includes_bill_or_action_identifier` | `legal_availability` |
| `lobbyist_reg_form_includes_position_on_bill` | `legal_availability` |
| `lobbyist_spending_report_includes_general_subject_matter` | `legal_availability` |
| `lobbyist_spending_report_includes_bill_or_action_identifier` | `legal_availability` |
| `lobbyist_spending_report_includes_position_on_bill` | `legal_availability` |

**Scoring rule:** form-agnostic OR over each concept-pair, then nested-tier table:
```
general_subject = reg_form_general_subject OR spending_report_general_subject
bill_id         = reg_form_bill_id         OR spending_report_bill_id
position        = reg_form_position        OR spending_report_position
```

| general_subject | bill_id | position | tier |
|---|---|---|---|
| F | F | F | −1 |
| T | F | F | 0 |
| T | T | F | 1 |
| T | T | T | 2 |

**Statutorily-inconsistent combinations** (e.g., `bill_id=T AND general_subject=F`, `position=T AND bill_id=F`) are implausible but representable. Decision: **flag as oddity in a structured warning return value, do not silently coerce**. Test fixtures must cover oddity combinations (see Testing Plan).

**Note on β AND-projection:** Sunlight item 1 introduces 3 spending-report-side rows (`lobbyist_spending_report_includes_{general_subject_matter, bill_or_action_identifier, position_on_bill}`) that **Opheim 1991 reads via β AND-projection** on bill_id AND position. The Opheim plan ([`20260514_opheim_1991_plan.md`](20260514_opheim_1991_plan.md), drafted same session) explicitly references these row IDs — keep these names stable.

### Item 2 — `sunlight_2015.expenditure_transparency` (4-tier, −1 to 2)

**Helper:** `project_sunlight_item2(cells: dict) -> int` returns `−1 | 0 | 1 | 2`.

**Reads 3 binary cells:**

| v2 row id | axis |
|---|---|
| `lobbyist_spending_report_required` | `legal_availability` |
| `lobbyist_spending_report_categorizes_expenses_by_type` | `legal_availability` |
| `lobbyist_spending_report_includes_itemized_expenses` | `legal_availability` |

**Scoring rule (clean nesting — each tier adds a requirement to the previous):**

| required | categorized | itemized | tier |
|---|---|---|---|
| F | * | * | −1 |
| T | F | F | 0 |
| T | T | F | 1 |
| T | * | T | 2 |

**Note:** Itemization implies categorization statutorily (itemization is finer than categorization), so `T, F, T` is implausible but representable. Treat as item-1-style oddity flag.

### Item 3 — `sunlight_2015.expenditure_reporting_thresholds` (2-tier, −1 to 0)

**Helper:** `project_sunlight_item3(cells: dict) -> int` returns `−1 | 0`.

**Reads 1 typed cell:**

| v2 row id | cell type | axis |
|---|---|---|
| `expenditure_itemization_de_minimis_threshold_dollars` | `Optional[Decimal]` | `legal_availability` |

**Scoring rule:** Sunlight reads presence/absence only:
```
threshold IS NULL OR threshold == 0  → 0   (all expenditures must be itemized)
threshold > 0                         → −1  (threshold exists; below-threshold lines exempt)
```

**Note on three distinct threshold concepts** (carry into module comment block — Sunlight #3 is the *itemization-line* threshold, NOT the registration or filing-de-minimis threshold; the v2 row is correctly scoped):
- **Lobbyist-status threshold**: `compensation_threshold_for_lobbyist_registration` (CPI #197 / HG Q2 / Newmark / Opheim / FOCAL scope.2)
- **Filing-de-minimis threshold**: separate PRI D1 row (whether registered lobbyist's activity triggers filing at all)
- **Itemization-de-minimis threshold**: `expenditure_itemization_de_minimis_threshold_dollars` ← **this row**

### Item 4 — EXCLUDED — `sunlight_2015.document_accessibility`

Do **not** implement. No helper, no test, no fixture. Module-level constant `EXCLUDED_ITEMS = frozenset({"sunlight_2015.document_accessibility"})` documents the exclusion; the test file asserts the exclusion is preserved (regression-guard against accidental implementation).

### Item 5 — `sunlight_2015.lobbyist_compensation` (2-tier, −1 to 0)

**Helper:** `project_sunlight_item5(cells: dict) -> int` returns `−1 | 0`.

**Reads 3 binary cells** (form-agnostic OR):

| v2 row id | axis |
|---|---|
| `lobbyist_spending_report_includes_total_compensation` | `legal_availability` |
| `lobbyist_spending_report_includes_compensation_broken_down_by_payer` | `legal_availability` |
| `lobbyist_reg_form_includes_compensation` | `legal_availability` |

**⚠ Rename applied:** spec doc says `_by_client`; v2 has `_by_payer`. See rename mapping table above.

**Scoring rule:**
```
compensation_disclosed_anywhere = total_on_spending_report
                                  OR broken_down_on_spending_report
                                  OR on_reg_form
→ 0 if disclosed; −1 otherwise.
```

## Module structure (mirror CPI 2015 C11)

Module path: `src/lobby_analysis/projections/sunlight_2015.py`.

**Module docstring shape** (copy from `pri_2010.py` and `cpi_2015_c11.py`):
- Module purpose (1 paragraph): rubric vintage, projection scope (item 4 excluded), validation regime (per-item only, no aggregation).
- Spec doc path.
- Ground-truth CSV path.
- Rename mapping table (the one above — `_by_client` → `_by_payer`).
- Conventions: α form-type split, β AND-projection (Opheim reads these rows), γ collect-once-map-many.

**Constants:**
- `EXCLUDED_ITEMS: frozenset[str] = frozenset({"sunlight_2015.document_accessibility"})`
- `LEGAL_AXIS: Final[str] = "legal_availability"` (mirror PRI 2010's axis constants if defined there).
- `IN_SCOPE_ITEMS: Final[tuple[str, ...]] = ("sunlight_2015.lobbyist_activity", "sunlight_2015.expenditure_transparency", "sunlight_2015.expenditure_reporting_thresholds", "sunlight_2015.lobbyist_compensation")`

**Pydantic score model:**
```python
class Sunlight2015Score(BaseModel):
    model_config = ConfigDict(frozen=True)
    state: str                                  # USPS code
    per_item_scores: dict[str, int]             # IN_SCOPE_ITEMS → signed tier (−2..2)
    oddity_flags: dict[str, list[str]]          # IN_SCOPE_ITEMS → list of oddity descriptions (empty list if none)
```

**No total, no grade, no rank.** These are not reproducible (item 4 excluded). The module **must not** expose `project_sunlight_2015_total` or `project_sunlight_2015_grade` or `rank_sunlight_2015_states`. Test for their absence.

**Top-level API:**
```python
def project_sunlight_2015(cells: dict, state: str) -> Sunlight2015Score: ...
def load_sunlight_2015_reference(repo_root: Path) -> dict[str, dict[str, int]]:
    """USPS → {item_id: signed_tier} reference from papers/Sunlight_2015__...csv.
    Strips footnote markers; preserves marker provenance in a sibling dict
    `load_sunlight_2015_reference_marker_provenance` if needed for caveat tests."""
```

**Per-item helpers:** `project_sunlight_item1` ... `project_sunlight_item5` (no item4). Each returns `int` in the appropriate signed range. Signature: `(cells: dict) -> int`. Cells parameter shape per CPI 2015 convention: `cells[row_id][axis] = value`. Helpers should also return oddity flags via a structured return (e.g., return a `(score, oddity_or_None)` tuple, threaded into `Sunlight2015Score.oddity_flags`).

**`unable_to_evaluate` convention** (per Sub-0 gap audit Pattern 3):
- Required input cells missing from `cells[row_id]` → helper returns `Literal["unable_to_evaluate"]` (NOT 0, NOT −1).
- Top-level `project_sunlight_2015` collects unable-to-evaluate items in `per_item_scores` as the string literal; aggregation is not attempted regardless.
- Tests must cover the missing-cell case for every helper.

## Edge cases (document in module + tests)

1. **Item 1 implausible combinations** (`bill_id=T AND general_subject=F`, etc.) — return computed tier + non-empty `oddity_flag`. Tests assert both the tier value AND the oddity flag presence.
2. **Item 2 itemization-without-categorization** (`T, F, T`) — same pattern as item 1.
3. **Item 3 zero-threshold vs null-threshold** — both treated the same (→ 0). Sunlight reads presence/absence only. Test both inputs explicitly.
4. **Footnote markers on per-state CSV cells** — strip before integer-coercion; preserve in marker-provenance dict; downstream tests verify provenance is recoverable. Markers carry uninterpretable caveat per `items_Sunlight.md` §7.
5. **Missing cells** — return `Literal["unable_to_evaluate"]` per item; tests cover each helper's missing-cell branch.
6. **State USPS code keys** — Sunlight CSV uses full state names; the reference loader must `STATE_NAME_TO_USPS[name]` (import from `scoring.calibration`) and key the dict by USPS code (mirror CPI / PRI convention).
7. **Casing on binary cells** — v2 TSV stores binary cells as raw strings; expect them as `"TRUE"`/`"FALSE"`/`"True"`/`"False"`/`"1"`/`"0"`/`""`. Coerce with the existing PRI / CPI helper if one exists in `scoring.calibration` or write a minimal `_to_bool(s: str | bool | None) -> bool | None` (returning `None` for empty string → `unable_to_evaluate` upstream).
