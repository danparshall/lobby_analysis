## Topics Explored

- Cell input shape for projection helpers — settled on
  `cells[row_id] = {"legal_availability": ..., "practical_availability": ...}`
  nested dict, harness-independent. Matches the spec's mental model.
- Per-item projection logic for 14 CPI 2015 C11 indicators — 6 de jure
  (Boolean / threshold / enum reads -> tier value) + 8 de facto (5-tier
  passthrough from the practical_availability axis). One compound
  passthrough (IND_205) reads 3 rows.
- Empirical aggregation-rule fit. Four candidates tried:
  simple mean, de-jure/de-facto halves, sequential sub-cats (2/4/3/2/3
  by IND order — the author's pre-extraction guess), and the actual CPI
  sub-cat structure extracted from `papers/CPI_2015__sii_criteria.xlsx`.
- Data-quality glitch resolution. Spec doc described "4 mixed-case
  typos + 2 numeric-where-categorical" cells. Counted 6 mixed-case
  (all in Colorado — 4 "Yes" + 2 "No") + 2 numeric ("100" in Texas
  IND_199 and Massachusetts IND_203). The empirical fit chose the
  normalization: case-insensitive YES/MODERATE/NO matching for the
  6 Colorado cells, and **NO (0) fallback** for the 2 numeric cells
  (CPI's aggregator treated them as invalid, not as YES).
- Rank tie-breaking. CPI 2015 uses **competition ranking** (1224 style:
  ties share a rank, the next rank skips). Sequential rank with
  alphabetical tiebreak (my first guess) was off-by-1 to off-by-2 on
  11 states; competition ranking is exact for all 50.

## Provisional Findings

- **The aggregation rule for CPI 2015 C11 is the unweighted mean of
  the 5 sub-category means** (sub-cats 11.1-11.5 with item counts
  2/4/3/2/3, extracted from `papers/CPI_2015__sii_criteria.xlsx`).
  Max abs residual across all 50 states is **0.05** — a one-decimal
  rounding artifact, not a fit error. The other three candidates
  (simple mean, de-jure/de-facto halves, sequential sub-cats) miss
  the tolerance on 18-38 of 50 states.
- The unweighted-sub-cat-means rule gives de jure items implicit
  higher per-item weight than de facto items: each of the 14 indicators
  contributes ``1 / (N_subcat_items * 5)`` to the category score, so
  the 2 items in sub-cat 11.1 each carry weight 0.1, while the 4 items
  in sub-cat 11.2 each carry 0.05. This is structurally what makes the
  fit work — the four de-facto items in 11.2 are partially
  discounted relative to the de-jure items in 11.1 and 11.3.
- The 2 numeric-string de jure cells (Texas IND_199 "100" and
  Massachusetts IND_203 "100") are best interpreted as Excel
  transcription artifacts that CPI's aggregator treated as **NO (0)**,
  not as YES (100). Evidence: setting these to NO produces exact-fit
  for both states (residual 0.05 from 1-dp rounding); setting them to
  YES produces +6.7 over-estimate for both states.
- CPI 2015 does **not** publish per-category letter grades — the `grade`
  column in `papers/CPI_2015__sii_scores.csv` is the overall-state
  grade, not per-category. The kickoff plan implied per-category
  grades; this is incorrect for C11. Letter-grade projection has no
  C11 ground truth to validate against and is dropped from this
  rubric's projection (likely also from PRI / Sunlight / Newmark /
  Opheim / HG / FOCAL).

## Decisions Made

- **Module layout.** `src/lobby_analysis/projections/<rubric>.py` +
  `tests/projections/test_<rubric>_*.py` (one file per concern:
  per-item, ground-truth-loader, aggregation+e2e). Matches the plan
  sketch.
- **Per-item helper return type.** Plain `int` in {0, 25, 50, 75, 100}.
  Avoids re-encoding the YES/MOD/NO tiers as enums when the canonical
  representation is the integer score.
- **Score type.** `CPI2015C11Score` Pydantic model carrying `state`,
  `per_item_scores: dict[str, int]`, `category_score: float`. No
  letter grade (not published per category). Frozen / immutable.
- **Cell input shape.** Nested dict: `cells[row_id][axis] = value`.
  Harness-independent.
- **De jure data-quality fallback.** Any de jure cell value not in
  YES/MODERATE/NO (after case-insensitive match) -> 0 (NO). Empirically
  fitted; comment in code explains derivation.
- **Rank tie-break.** Competition ranking (1224), not sequential.
- **Aggregation rule documentation.** Hardcoded in
  `aggregate_cpi_2015_c11` with provenance comment pointing at
  `scripts/fit_cpi_2015_c11_aggregation.py`. Fit script reproduces the
  decision and evaluates alternative candidates.

  implement 8 of these projection modules. The current
  per-item-helper-per-indicator pattern is verbose (14 functions for
  CPI; PRI 2010 will be ~50). Consider a declarative table for

  cost will be concrete.

  per-category grades. Other rubrics (PRI / Newmark / etc.) likely
  also don't publish per-category grades; confirm rubric-by-rubric.
- **Cross-rubric agreement audit** (Phase 4 of the plan). After all 8
  rubrics ship, the test is "for each multi-rubric row, do all
  consuming projections produce the same tier for a sample of states."
  Pull this into a single shared test once two projection modules
