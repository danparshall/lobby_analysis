# Opheim 1991 Projection Implementation Plan

**Goal:** Ship `project_opheim_1991(state, cells)` — a per-state projection function that reads 14 v2 compendium row families (all pre-existing) and produces Opheim 1991's per-state disclosure-side partial score, validated against the published Table 1 totals via weak inequality.

**Originating conversation:** [`../convos/20260514_rubric_plans_drafting.md`](../convos/20260514_rubric_plans_drafting.md) (Sub-0 of 5 — playbook gap audit + data-year audit). This plan was drafted in Sub-1 (Stream 1 plans) alongside [`20260514_sunlight_2015_plan.md`](20260514_sunlight_2015_plan.md).

**Context:** Rubric #6 in Phase C's locked rubric order, but **Stream 1's second plan** — it depends on Sunlight 2015's α form-type split having locked the bill_id + position row IDs that Opheim's β AND-projection reads. Opheim is the **earliest contributing rubric** (1988–89 vintage, published 1991) and the explicit predecessor of Newmark 2005; its Phase C role is **temporal-depth validation** of the BoS-derived definitional-and-disclosure backbone, extending the contributing-rubric set to ~28 years (1988-89 → 2015). 14 of 14 row families touched are pre-existing — **zero new rows introduced**.

**Confidence:** High on per-item mappings (spec doc factual-audit-clean; all 14 row families pre-existing from CPI / PRI / Sunlight / Newmark 2017 mappings). High on data year (paper-explicit, 1988-89 CSG Blue Book). **Medium on validation utility** — weak inequality `our_partial ≤ paper_total` is the only available Phase C check (Opheim publishes per-state totals only; no sub-aggregates, no per-item).

**Architecture:** Declarative `_ATOMIC_SPEC` table dispatcher with one named-helper exception (the cadence 2-cell OR; the β AND-projection). Mirrors `src/lobby_analysis/projections/pri_2010.py`'s pattern — 11 of 14 in-scope items are pure binary-cell → 0/1 passthroughs, 3 typed-cell `IS NOT NULL` reads, 1 cadence-OR helper, 1 bill_id+position AND helper, 1 un-projectable catch-all. **No `index.total` aggregation function** — projection cannot reproduce Opheim's 0–22 published total (8 items OOS or un-projectable). Section sub-aggregates are produced as informational outputs but aren't validated (Opheim doesn't publish them).

**Branch:** `phase-c-projection-tdd` (worktree at `.worktrees/phase-c-projection-tdd`).

**Tech Stack:** Python 3.12, Pydantic v2, pytest, `uv` for env management. `load_v2_compendium()` from `lobby_analysis.compendium_loader` for cell access.

## Validation regime — Weak-inequality only

**Tier:** Weak-inequality only (per Sub-0 gap audit Pattern 2 third tier).

**Ground truth:** Opheim 1991 paper Table 1 (paper line 174 caption; 47-state ranking through ~line 200). Per-state `index.total` only. **No sub-aggregate breakdown published. No per-atomic-item per-state data published.**

**Sample:** 47 states (Montana, South Dakota, Virginia excluded per paper line 176: "Data for Montana, South Dakota, and Virginia were unavailable").

**Available Phase C checks:**
- `our_projected_partial[state] <= paper_total[state]` for all 47 in-sample states. The 7 OOS `enforce.*` items + the 1 un-projectable catch-all contribute `∈ [0, 8]` to `paper_total` but `0` to `our_projected_partial`; the inequality must hold.
- **0** cells usable for sub-aggregate tolerance check (paper doesn't publish sub-aggregates).
- **0** cells usable for per-atomic-item check (paper doesn't publish per-item per-state data).

**Validation tolerance:** `<=`. No equality check available.

**Cross-rubric validation is the actual quality check.** Opheim's BoS-sourced atomic items are correct-by-construction when the same compendium cells are validated against PRI 2010 per-item data (`docs/historical/pri-2026-rescore/`), CPI 2015 C11 per-state data (`docs/historical/compendium-source-extracts/results/cpi_2015_c11_per_state_scores.csv` and `papers/CPI_2015__sii_scores.csv`), Newmark 2017 sub-aggregates (when shipped), and Sunlight 2015 per-item data. Phase 4 cross-rubric audit (deferred until ≥3 modules ship) is where Opheim's contribution becomes load-bearing.

**Plan-honesty note (do NOT promise what the regime can't deliver):** Tests **must not** assert `abs(our_partial - paper_total) <= 1` or any equality-flavored tolerance. Only `<=` may be asserted on the partial-vs-total relationship.

### Disclosure battery (7 in-scope projectable items + 1 un-projectable)

#### `opheim_1991.disclosure.frequency` (cadence 2-cell OR — named helper)

**Helper:** `project_opheim_disclosure_frequency(cells: dict) -> int` returns `0 | 1`.

**Reads 2 binary cells** (subset of the 8-cell PRI cadence row family):

| v2 row id | axis |
|---|---|
| `lobbyist_spending_report_cadence_includes_monthly` | `legal_availability` |
| `principal_spending_report_cadence_includes_monthly` | `legal_availability` |

**Scoring rule:** `(lobbyist_monthly OR principal_monthly) → 1; else → 0`.

**⚠ Rename applied:** spec doc says `_report_cadence_includes_monthly`; v2 has `_spending_report_cadence_includes_monthly` (both lobbyist + principal). See rename mapping table above.

**Note on the monthly-only cut:** Opheim's binary collapse is FINER than Newmark 2005's. Same underlying row family (PRI E1h_* / E2h_*, 8 binary cells); Opheim reads 2 of 8 cells; Newmark 2005 reads 8 of 8. A state with quarterly reporting gets `Newmark 2005 = 1` (quarterly counts as >annual) but `Opheim = 0` (not monthly). **Both are correct projections of their respective rubric's published rule.** This is the canonical example of Sub-0 gap audit Pattern 4 ("same-row-different-binary-cut") — document explicitly in the helper docstring.

**Note on tri-annual treatment:** Opheim's source quote (paper line 117) enumerates "quarterly, semi-annually, or annually were coded 0" and omits tri-annual. **Strict interpretation: tri-annual → 0** (less frequent than monthly). The helper reads only the monthly cells; tri-annual cells are not read; tri-annual states score 0.

**Note on session-bounded vs always-monthly** (Open Issue 1 in spec doc, deferred to compendium 2.0 freeze): PRI's `_cadence_includes_monthly` cell doesn't distinguish "monthly during session" from "monthly in-and-out-of-session." Both Opheim treats as 1. **The projection accepts the coarser cell as-is; if a future rubric needs the distinction, the cell will need splitting at the schema layer.** Not in scope for this plan.

## Module structure (mirror PRI 2010 — declarative table dispatcher)

Module path: `src/lobby_analysis/projections/opheim_1991.py`.

**Module docstring shape** (copy from `pri_2010.py`):
- Module purpose (1 paragraph): rubric vintage (1988-89), projection scope (7 enforce + 1 un-projectable OUT), validation regime (weak-inequality only).
- Spec doc path.
- Ground-truth source: Opheim 1991 paper Table 1 (47-state ranking, paper line 174 ff). **No CSV exists — Phase 0 task extracts Table 1 to a generated CSV.**
- Rename mapping table (4 renames documented above).
- Conventions: β AND-projection (2nd concrete use after Sunlight), `unable_to_evaluate` convention, 47-state sample (MT/SD/VA missing).

**Constants:**
```python
OOS_ITEMS: frozenset[str]               # 7 enforce.*  (disclosure-only Phase B)
UNPROJECTABLE_ITEMS: frozenset[str]     # 1 catch-all
IN_SCOPE_PROJECTABLE_ITEMS: tuple[str]  # 14 items in fixed order (def first, then disclosure)
DEF_SECTION_ITEMS: tuple[str]           # 7 def.* items
DISCLOSURE_SECTION_ITEMS: tuple[str]    # 7 in-scope disclosure items (1 freq + 6 categories; catch-all not here)
SAMPLE_MISSING_STATES: frozenset[str]   # frozenset({"MT", "SD", "VA"})
LEGAL_AXIS: Final[str]
```

**Declarative spec table** (mirror PRI's `_ATOMIC_SPEC` dict):
```python
# {opheim_item_id: SpecEntry}
# SpecEntry is either:
#   ("binary_true", row_id, axis)                — most items
#   ("typed_is_not_null", row_id, axis)          — 3 def.*_standard threshold cells
#   ("cadence_or_monthly", None, None)           — dispatched to named helper
#   ("gifts_or_actor_agnostic", None, None)      — dispatched to named helper
#   ("legislation_and_bill_position", None, None) — dispatched to named helper (β)
#   ("unable_to_evaluate", None, None)           — catch-all
```

**Named helpers:** `project_opheim_disclosure_frequency`, `project_opheim_disclosure_gifts`, `project_opheim_disclosure_legislation`. Each reads 2 cells (`legal_availability` axis), returns `int` in `{0, 1}` or the literal `"unable_to_evaluate"` if any required cell missing.

**Pydantic score model:**
```python
class Opheim1991Score(BaseModel):
    model_config = ConfigDict(frozen=True)
    state: str                                    # USPS code; must NOT be in SAMPLE_MISSING_STATES
    per_item_scores: dict[str, int | str]         # 14 projectable items → 0/1; 1 catch-all → "unable_to_evaluate"
    unevaluable_items: list[str]                  # contains the catch-all item id (always 1 entry under current plan)
    def_section_total_partial: int                # sum over DEF_SECTION_ITEMS, ∈ [0, 7]
    disclosure_section_total_partial: int         # sum over DISCLOSURE_SECTION_ITEMS (7 items; catch-all excluded), ∈ [0, 7]
    total_partial: int                            # def + disclosure, ∈ [0, 14]
    # NO total (cannot reproduce; OOS items not projected).
    # NO rank (Opheim's published rank is over 22-point total; we can't reproduce).
```

**`SAMPLE_MISSING_STATES` enforcement:** `project_opheim_1991(cells, "MT")` raises `ValueError("Opheim 1991 has no published score for state MT (paper line 176)")`. Same for SD and VA. Tests cover this.

**Top-level API:**
```python
def project_opheim_1991(cells: dict, state: str) -> Opheim1991Score: ...
def load_opheim_1991_reference(repo_root: Path) -> dict[str, int]:
    """USPS → index.total for the 47 in-sample states.
    Loaded from a generated CSV at `data/opheim_1991_index_totals.csv` (Phase 0 task);
    47 entries; MT, SD, VA absent."""
```

**No aggregation function:** `project_opheim_1991_total` does **not** exist (Opheim's 22-point total cannot be reproduced from 14 projected items; constructing one would smuggle a wrong-by-design API). Test for its absence.

## Edge cases (document in module + tests)

1. **`TimeThreshold` typed-cell as raw string** — v2 loader returns strings; `IS NOT NULL` check is a non-empty-string check. If `extraction-harness-brainstorm` ships typed models later, the helper logic doesn't change. Document in helper docstring.
2. **String `"0"` for compensation threshold** — non-empty string; per `IS NOT NULL` semantics this means the threshold is `$0`, which is **a threshold that exists** (lobbyist-status triggers at $0 of compensation). Score = 1 per Opheim's rule. Verify intent: grep `pri_2010.py` for existing `_is_not_null` helper and reuse if available; otherwise write a minimal one and use it.
3. **Casing on binary cells** — same convention as Sunlight plan. Reuse PRI 2010's `_to_bool` helper if available; otherwise write a minimal `_to_bool(s: str | bool | None) -> bool | None` returning `None` for empty string → `unable_to_evaluate` upstream.
4. **MT/SD/VA refusal** — projection raises `ValueError` rather than returning a sentinel. Reason: silent zero would corrupt weak-inequality check; sentinel `"unable_to_evaluate"` muddles the per-state-coverage signal. Hard refusal is the cleanest.
5. **Catch-all in `disclosure_section_total_partial` sum** — must be excluded. Section total covers 7 disclosure items (1 frequency + 6 categories), not 8. Test asserts `disclosure_section_total_partial <= 7`.
6. **Tri-annual cadence** — not read by Opheim's helper. Setting tri-annual cells to TRUE does NOT contribute to the cadence projection score. Test fixture explicitly verifies this.
7. **Cross-rubric continuity** — Opheim's β AND on bill_id + position reads the SAME compendium cells as Sunlight item 1 reads at finer granularity. The continuity test pins this invariant; if either Sunlight or Opheim drifts on row names, the cross-rubric test catches it.
8. **State USPS keying** — Opheim Table 1 uses full state names; ground-truth loader converts via `STATE_NAME_TO_USPS` and keys by USPS. Mirror CPI / PRI / Sunlight convention.
