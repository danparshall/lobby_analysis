## Architecture

- **Module:** `src/lobby_analysis/projections/pri_2010.py` (mirrors CPI 2015 C11 module location).
- **Per-item layer:** declarative `_PRI_ITEM_SPECS` table mapping each PRI atomic item ID → (v2 compendium row id, axis, cell type). 75 of 76 items are pure binary passthroughs; only Q8 is typed 0..15 (passthrough divided by 15). The declarative table resolves the verbosity tension flagged as Open Question in CPI 2015's convo — function-per-item was the right choice for CPI's 14 items but doesn't scale to PRI's 76.
- **Rollup layer:** imports `rollup_disclosure_law` / `rollup_accessibility` from `src/scoring/calibration.py`. No new rollup code.
- **Score models:** `PRI2010DisclosureLawScore` (state + atomic_scores + sub_aggregates + total + percent + rank), `PRI2010AccessibilityScore` (state + atomic_scores + sub_components + total + percent + rank). No letter grade — PRI 2010 doesn't publish one.

## Decisions made

- **Module location.** `src/lobby_analysis/projections/pri_2010.py` (mirrors CPI 2015 C11 module location).
- **Per-item layer architecture.** Declarative spec table + single dispatching helper, not function-per-item. This is the refactor the CPI session's Open Question deferred to rubric #2.
- **Two top-level functions.** `project_pri_2010_disclosure_law(cells, state)` returns `PRI2010DisclosureLawScore`; `project_pri_2010_accessibility(cells, state)` returns `PRI2010AccessibilityScore`. Mirrors PRI's two-axis published structure.
- **Reuse `scoring.calibration` rollup helpers.** No rebuild; the existing implementation is paper-derived, line-cited, and unit-tested. The projection module imports `rollup_disclosure_law`, `rollup_accessibility`, `STATE_NAME_TO_USPS`, and the `DisclosureSubAggregates`/`AccessibilitySubAggregates` Pydantic models.
- **Skip 7 atomic items not read by the rollup** (C1, C2, C3, D1_present, D1_value, D2_present, D2_value). Paper scores C/D as 0/1 gates only; sub-criteria are descriptive. YAGNI — add helpers when a consumer needs them.
- **Cell input shape** carries forward from CPI: nested `cells[row_id][axis] = value`, harness-independent.
- **No letter grade.** PRI 2010 does not publish per-category letter grades (CSV columns: state, sub-aggregates, total, percent, rank — no grade column). Same pattern as CPI 2015 C11. Score models drop the letter-grade field.
- **Rank.** Competition (1224) ranking, verified to reproduce PRI's published `rank_2010` column exactly.
- **PRI-MVP retirement.** Source moved (not deleted) to `src/scoring/_deprecated/smr_projection.py` with SUPERSEDED banner; tests moved to `tests/_deprecated/test_smr_projection.py`. Default pytest collection excludes `_deprecated/` via `norecursedirs`. The v1 PRI-MVP pipeline remains recoverable.
